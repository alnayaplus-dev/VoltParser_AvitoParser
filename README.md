### 🚀 VoltParser

VoltParser — это браузерное расширение (Chrome / Edge) для **парсинга данных с Avito** прямо со страниц сайта. Оно не использует внешние прокси и работает из вашего браузера, имитируя действия реального пользователя. Расширение состоит из нескольких модулей, каждый из которых отвечает за свой режим сбора данных.

---

## 🧠 Архитектура и файлы

| Файл | Назначение |
| --- | --- |
| `manifest.json` | Манифест расширения (версия 3), права доступа (`activeTab`, `scripting`, `host_permissions: avito.ru`), точки входа. |
| `background.js` | Фоновый сервис-воркер: обрабатывает клик по иконке расширения и отправляет команду `togglePanel` на активную вкладку. |
| `utils.js` | Общие утилиты: задержки (`randomSleep`), повторные запросы (`fetchWithRetry`), скроллинг (`randomScroll`), скачивание CSV (`downloadCSV`), стоп-слова (`STOP_WORDS`), проверки слов. |
| `content.js` | Главный скрипт внедрения. Создаёт плавающую панель с кнопками, управляет запуском/остановкой режимов, вызывает модальные окна из других файлов. |
| `phrases.js` | Модальное окно «Фразы»: загрузка исходных фраз (ручной ввод или CSV), генерация комбинаций (подпоследовательности с первым словом, от 2 слов, мин. длина слова 4), ограничение топ-500 самых частотных. |
| `demand.js` | Модальное окно «Анализ спроса»: загрузка списка фраз, кнопка массового открытия страниц `avito.ru/analytics/wordstat` для каждой фразы (пользователь сам выбирает категорию и нажимает «Показать»). |
| `semantic.js` | Модальное окно «Семантика»: парсинг заголовков и описаний объявлений, сбор частоты слов (по отдельности для заголовков и описаний) и фраз из 2–5 слов, фильтрация (минимальная длина слова 4, частота ≥2, стоп-слова, числа), выгрузка в один CSV. |

Все файлы загружаются в контент-скрипт в порядке: `utils.js` → `phrases.js` → `demand.js` → `semantic.js` → `content.js`.

## 🛡️ Защита от бана (механизмы `utils.js`)

Расширение эмулирует человеческое поведение, чтобы не быть заблокированным Avito:

1. **Случайные задержки** (`randomSleep`) — от 200 мс до нескольких секунд между запросами.
2. **Повторные попытки при ошибках** (`fetchWithRetry`):
    - Если сервер вернул `429` (Too Many Requests) — ждёт 5–7 секунд и повторяет (до 2 раз).
    - При любой другой ошибке — экспоненциальная задержка (1, 2, 4 секунды).
3. **Скроллинг страницы** (`randomScroll`) — случайный скролл до начала парсинга для имитации просмотра.
4. **Паузы каждые 5–7 объявлений** — 6–12 секунд ожидания и скролл, чтобы сбросить счётчик запросов.

## 📊 Режим «Статистика» (`content.js`)

**Что делает:**

Собирает с каждой открытой страницы списка объявлений ссылки на карточки товаров (по атрибуту `[itemprop="url"]`). Затем последовательно открывает каждую карточку (через `fetch`), парсит HTML и извлекает:

- **Заголовок** — из `h1`.
- **Описание** — по нескольким селекторам (`[itemprop="description"]`, `[data-marker="item-description"]`, либо из скрипта c полем `"description"`).
- **Просмотры (всего)** — регулярное выражение `(\\d[\\d\\s]*)\\s+просмотр`.
- **Просмотры (сегодня)** — `\\+?\\s*(\\d+)\\s+сегодня`.
- **Категория** — из URL `avito.ru/[регион]/[категория]`.
- **Ссылка** — оригинальный URL.

**Выходной файл:** `stats_full.csv` с колонками:

`title;description;views_total;views_today;category;url`

**Защита:**

После каждых 5 объявлений встаёт пауза 6–12 секунд + скролл. На каждую карточку — задержка 1–4 секунды. При ошибке — ждёт 3–6 секунд.

## 📈 Режим «Фразы» (`phrases.js`)

**Что делает:**

Позволяет **загрузить список исходных фраз** (вручную или из CSV, берётся **только первый столбец**) и **сгенерировать из них комбинации** — все возможные подпоследовательности, начинающиеся с первого слова и содержащие **не менее 2 значимых слов**.

**Алгоритм генерации комбинаций:**

1. Текст фразы разбивается на слова по пробелам.
2. Каждое слово очищается от знаков препинания (остаются только буквы/цифры).
3. Отбрасываются слова короче 4 символов (`MIN_WORD_LEN = 4`).
4. Из оставшихся слов берётся **первое слово** как обязательное начало.
5. Для набора остальных слов перебираются все непустые подмножества (битовая маска). Каждое подмножество добавляется после первого слова → итоговая комбинация.
6. **Частота** комбинации = сколько раз она встретилась среди всех исходных фраз.
7. В конце сортировка по убыванию частоты, берутся **топ-500 самых популярных**.

**Пример:**

Исходная фраза: `"зеркала бу круглые"`

Значимые слова: `"зеркала"`, `"бу"`, `"круглые"` (все ≥4) → генерация:

- `"зеркала бу"`
- `"зеркала круглые"`
- `"зеркала бу круглые"`

**Выходной файл:** (кнопка «Скачать CSV») – `phrases.csv` с колонкой `phrase`.

**Ограничения:**

- Не более 2000 исходных фраз (предупреждение).
- В процессе генерации все комбинации хранятся в `Map` (комбинация → частота). При большом количестве (десятки тысяч) браузер может замедлиться, но не крашится, т.к. финальное ограничение 500 снижает нагрузку.

**Прогресс-бар** показывает текущую исходную фразу и накопленное количество уникальных комбинаций.

## 🔍 Режим «Семантика» (`semantic.js`)

**Что делает:**

Анализирует **все объявления** из списка (по ссылкам на карточки). Для каждого объявления парсит **заголовок** и **описание** (аналогично режиму «Статистика»). Затем:

1. Разделяет текст на слова (только буквы/цифры, приведённые к нижнему регистру).
2. **Отдельно** ведёт статистику для слов из заголовков и для слов из описаний.
3. Также выделяет из **объединённого текста** (заголовок + описание) **фразы длины 2, 3, 4, 5 слов** (скользящее окно) — отдельно для заголовков и для описаний.
4. **Фильтры** (применяются к словам и к каждому слову внутри фразы):
    - Частота встречаемости **≥2**.
    - Длина слова **≥4 символов**.
    - Слово **не входит** в стоп-список (предлоги, союзы, местоимения, типичные слова-паразиты «продам», «новый», «вкл», «выкл» и т.д.).
    - Слово не является числом (только цифры).
5. **Сортировка** по убыванию частоты отдельно для слов и для фраз.

**Выходной файл:** `semantic_words.csv` с четырьмя колонками:

```
word_title;freq_title;word_desc;freq_desc
слово1; 3; словоA; 10
слово2; 2; словоB; 5
...
фраза1 (2‑5 слов из заголовка); частота; фраза1 (из описания); частота
...
```

Фразы идут **подряд без разделителей** после однословных слов (поскольку заголовок колонок одинаковый).

**Защита от бана:**

Аналогично режиму «Статистика»: задержки между карточками, паузы после каждых 7 объявлений, скроллинг.

## 📈 Режим «Анализ спроса» (`demand.js`)

**Что делает:**

Позволяет **загрузить список фраз** (ручной ввод или CSV) и **массово открыть** страницы `https://www.avito.ru/analytics/wordstat?q=фраза` в новых вкладках. Пользователь затем вручную выбирает категорию и нажимает «Показать» — расширение **не автоматизирует** заполнение формы (это было бы нарушением правил Авито и привело бы к бану).

**Интерфейс:**

- Текстовое поле для ввода/редактирования фраз.
- Кнопка «Загрузить CSV» (первый столбец).
- Кнопка «Открыть все фразы в Wordstat» — каждая фраза в отдельной вкладке.
- Предпросмотр списка с индивидуальными кнопками «Открыть Wordstat» для каждой фразы.

**Ограничения:**

- Страница `/analytics/wordstat` требует авторизации в Avito как продавец и подключенного тарифа с аналитикой.
- Расширение не парсит данные с этой страницы, только открывает её.

## 💻 Установка и использование

1. Скачайте папку с расширением (все файлы: `manifest.json`, `background.js`, `utils.js`, `phrases.js`, `demand.js`, `semantic.js`, `content.js`, иконка `icon1024.png`).
2. В Chrome перейдите на `chrome://extensions/`, включите **«Режим разработчика»**.
<img width="1908" height="870" alt="image" src="https://github.com/user-attachments/assets/98d41bc5-728c-4ebb-b442-f2c8b448b8b1" />
3. Разархивируйте .ZIP файл, нажмите **«Загрузить распакованное»** и выберите разархивированную папку
<img width="1906" height="867" alt="image" src="https://github.com/user-attachments/assets/4f8d5072-61fb-41b2-be72-866b96a0da6c" />
4. После установки нажмите на иконку расширения на любой странице `avito.ru` (список объявлений).
5. Появится панель с кнопками:
    - **📊 Статистика** — запускает сбор полных данных.
    - **📈 Фразы** — открывает модальное окно для генерации комбинаций.
    - **📈 Анализ спроса** — открывает модальное окно для работы с Wordstat.
    - **🔍 Семантика** — запускает частотный анализ.
6. Для остановки долгого процесса (например, при парсинге сотен объявлений) повторно нажмите ту же кнопку — она превратится в **⏹ Стоп**.

---

## ⚠️ Важные замечания

- Расширение работает **только на `avito.ru`**.
- Все запросы идут с вашего IP — **не парсите тысячи объявлений** за раз, иначе Avito может временно заблокировать доступ.
- Страница аналитики Wordstat требует авторизации и платного тарифа. Расширение лишь открывает её, но не вводит данные автоматически.
- При парсинге 100+ объявлений процесс может занять 10–20 минут из-за встроенных задержек (защита от бана).
- Если вам нужно изменить минимальную длину слова (сейчас 4) или количество топ-комбинаций (сейчас 500) – отредактируйте соответствующие константы в `phrases.js` и `semantic.js`.

## 🔧 Возможные доработки (для разработчика)

- Изменить `MIN_WORD_LEN` в `utils.js` (фильтр слов по длине) и в `phrases.js`.
- Изменить `TOP_N` в `phrases.js` (количество итоговых комбинаций).
- Изменить стоп-слова в `utils.js` (добавить/убрать).
- Добавить поддержку других сайтов (требует изменения селекторов и логики).

Если нужны дополнительные пояснения или вы хотите расширить функционал — обращайтесь.
